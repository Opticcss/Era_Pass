开头 | erapass 建站手册

今晚站点的编译环境崩了 | 一周努力白费 | 重开的同时留个念想 | 方便以后从头再来.
views | comments

当前个人网站经过了多次迭代, 是典型的静态网页, 依靠本地编译生成, 并基于 GitHub Pages ↗ 进行部署. 这类网站完全免费且不需要任何数据库支持, 但具有如下几个缺点,

  • 占用空间限制: space should not be greater than 1 G.
  • 流量限制: monthly flow should not be greater than 100 G.
  • 更新次数限制: cannot be updated 10 times per hour.
  • 自定义功能限制: cannot be equipped with comment function (this can be complemented by third-party services).

尽管如此, 静态网页带来了更多好处, 成为个人网站项目的首选.

注意, 静态网页只由 plain HTML 文件构成, 不同于由 PHP 生成的动态网页, 静态网页对于每个用户都展示相同的界面. 这也帮助没有数据库编程经验的人仅依靠 HTML/CSS 知识完成建站.

此类网站一般加载得更快, 因为服务器不需要解析任何内容, 只是读取 plain HTML 文件. 其与动态网页的优劣区别需结合实际应用权衡.

网站选取 Astro ↗ 高效能网络框架作为静态站点生成器 (static site generator, SSG), 它是首个普及群岛架构 (islands architecture) 的框架, 大幅减少了用户端负担从而提升网页的初始加载性能, 开发者可在 Astro 中对同一专案同时使用如 React ↗ 和 Vue ↗ 等多种 UI 框架.

个人网页平台利用 markdown 以及 Markdown for the component era ↗ (MDX) 编写的文章生成对应网页, 并支持通过 Vite 使用如 Sass 等 CSS 预处理器, 将脚本解析成 CSS. Astro 可以看作一类内容管理系统 (content management system, CMS, 可作 client 使用), 能够加载自定义 layout 文件以及对应 .md 与 .mdx 文件到每一网页中, 从而节约重写次数.

具体地, 本项目基于 Astro 构建, 支持 Markdown, MDX, 组件化开发, 并集成搜索, 标签, 评论等功能, 以下为项目文件结构.

  • src/ 源代码主目录. 用户的主要工作区, 包含页面, 组件, 内容和样式.

    • src/pages/ 页面路由. 每个 .astro、.md、.mdx 文件会自动生成对应的路由 (如 pages/about/index.astro →\to→ /about).
    • src/content/ 内容集合. 包含博客 (blog/, 每篇文章可包含标题, 日期, 标签等 Frontmatter), 其它文档等.
    • src/components/ 可复用 UI 组件. 按功能模块划分为 about/, home/, links/, projects/, waline/ 评论组件 (文件类型 .astro/.jsx/.tsx).
    • src/layouts/ 页面布局. 定义全局页面结构 (头部, 底部, 侧边栏).
    • src/assets/ 静态资源. 图片, 字体, 图标, 样式文件等.

  • public/ 公共静态资源. 直接映射到网站根目录的 /favicon/, /images/, /scripts/ 等资源.

  • dist/ 构建输出目录. 为运行 npm run build 后生成的用于部署的静态文件 (包含所有页面, 资源, 搜索索引等).

  • packages/pure/ 本地核心模块, 与主项目解耦.

  • node_modules/astro-pure 主项目核心模块.

  • astro.config.ts Astro 根目录配置文件. 用于配置站点 URL, 构建选项, 集成, Vite 设置等.

① 云端分支部署#

手动将项目编译得到的静态网页 dist 上传到 GitHub 中, 进入仓库 Settings-GitHub Pages-Build and deployment-Branch , 选择 jinzhao_release 作为当前站点的 source.

首先运行编译命令, 同时本地预览.

npm run build
npx astro preview
bash

实时更改的本地预览命令为.

npm run dev
bash

同样于链接 http://localhost:4321/Era_Pass/ 检测生成网页. 检查后变更网络至代理 (如 Clash 常用端口 7890), 并将生成静态网页推送至 Github 云端分支 jinzhao_release (生成 dist 静态网站目录剥离为本地临时分支 deploy-branch, 再覆盖到云端分支).

git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
git add dist -f
git commit -m "Deploy: static site content update"
git subtree split --prefix dist -b deploy-branch
git push origin deploy-branch:jinzhao_release --force
bash

网页正确运行则将在仓库 Action 页面出现蓝色 check mark, 否则显示红色 error mark, 成功托管的静态网页可在如下网址 erapass ↗ 访问.

Jinzhao Portfolio

Mapping personal academic research, insights, life, and my signals onto the time-frequency plane. Dedicated to fellow signal processing enthusiasts. (opticcss.github.io)

Card 背景框格功能
